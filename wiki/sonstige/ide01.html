Eine IDE ist eine integrierte Entwicklungsumgebung. Das heißt, dass möglichst viele Aspekte der Softwareentwicklung in einem Programm (oder einem Set von Programmen) dargestellt werden können. Eine solche IDE ist Microsoft Visual Studio Code, die wir hier benutzen. Im Endeffekt könnte man coden mit einem Texteditor (vom eingebauten Notepad bis hin zu aufgebohrten Texteditoren wie Atom oder Notepad ++) und einem Compiler, wenn es sich um compilierten Code handelt. Desto eher der Texteditor, der sicher weiter im Kern vorhanden ist, aufgebohrt wurde, desto eher ist es eine IDE (statt einem Texteditor). Am Beispiel von VSCode sieht man dann deutlich, wie sehr Features hinzu gekommen sind, die es von einem normalen Texteditor unterscheiden: * Ganze Ordnerstrukturen werden verwaltet statt einzelner Textfiles * Integration von GIT-Versionsverwaltung, inkl. direkter Anbindung an Github * Integration von etlichen Sprachen (via Plugins) * Direkte Kompilierung aus VSCode heraus (in vielen Sprachen) * Syntax-Highlighting (farbige Darstellung von Befehlen etc.) und Linter-Regeln (das sind die blauen Linien, die wir oft im Code sehen) * Direktes Debugging aus der IDE heraus (wo man in der Laufzeit der App z.B. die Belegung von Variablen sehen kann) * Unterstützung etlicher Dateitypen, auch Grafiken etc. Das alles zusammen macht dann eine gute IDE aus, und im Endeffekt ist die Entscheidung für eine bestimmte Geschmackssache.
